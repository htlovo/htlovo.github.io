正则表达式标志(g,i,m)
	g	全局，匹配多次
	i	不区分大小写
	m	多行

RegExp实例的属性
	global	布尔值，表示是否设置g标志
	ignoreCase	布尔值，表示是否设置i标志
	lastIndex	下一次exec匹配开始的索引。初始值为0
	multiline	布尔值，表示是否设置m标志
	source		正则表达式字符串表示
RegExp实例的方法
	exec()	接受一个字符串参数，返回包含第一个匹配项信息的数组或null。
		返回的数组包含额外两个属性
			index	匹配项在字符串中的位置
			input	应用正则表达式的字符串
		数组第一项是与整个模式匹配的字符串，其他项是与模式中的捕获组匹配的字符串
		在全局模式下每次调用exec()都会返回字符串中的下一个匹配项，直至搜索到字符串末尾为止，lastIndex每次调用在exec()后都会增加
	test()	接受一个字符串参数，如果匹配成功返回true，否则返回false
	compile()	可用于改变和重新编译正则表达式。
	继承的方法：toLocaleString()，toString()方法都会返回正则表达式字面量，valueOf()方法返回正则表达式自身，

RegExp构造函数属性
	属性适用于作用域中的所有正则表达式，并且基于所执行的最近一次正则表达式操作而变化
	input			$_	最近一次要匹配的字符串
	lastMatch		$&	最近一次的匹配项
	lastParen		$+	最近一次匹配的捕获组
	leftContext		$`	input字符串中lastMatch之前的文本
	multiline		$*	布尔值，表示是否所有表达式都使用多行模式
	rightContext	$'	input字符串中lastMatch之后的文本
					$1	用于存储第一个匹配的捕获组
					$9	用于存储第九个匹配的捕获组
					
/	正则表达式开始、结束标志

|	分隔正则表达式序列，符合任何一个序列匹配条件即被匹配

.	匹配除结束符外的任何字符
^	匹配文本的开始
$	匹配文本的结束

{3,6}	因子被匹配3、4、5或6次
？	匹配0或1次，等同于{0,1}
*	匹配0或多次，等同于{0,}
+	匹配1或多次，等同于{1,}
量词后加？进行非贪婪匹配


( )		捕获型分组
(?: )	非捕获型分组
(?= )	向前正向匹配
(?! )	向前负向匹配

[ ]		字符集，匹配里面有的字符
[^ ]	匹配里面没有的字符
-		包扩前后两个字符和它们之间的字符


正则表达式转义
	\f	换页符
	\n	换行符
	\r	回车符
	\t	制表符
	\u四个十六进制数字	Unicode字符
	\d	匹配一个数字，等同于[0-9],\D相反
	\s	Unicode空白符的一个不完全子集，等同于 [\f\n\r\t\u000B\u0020\u00A0\u2028\u2029]，\S相反
	\w	话语字符，等同于[0-9A-Z_a-z],\W相反
	\b	匹配文本的字边界，\B相反
	\1	指向分组1所捕获的文本的一个引用，\2,\3以此类推
	必须被转义才能匹配字面量的字符：\/[](){}?+*|.^$

正则表达式字符转义
	\b	退格符
	\f	换页符
	\n	换行符
	\r	回车符
	\t	制表符
	\u四个十六进制数字	Unicode字符
	\d	匹配一个数字，等同于[0-9],\D相反
	\s	Unicode空白符的一个不完全子集，等同于 [\f\n\r\t\u000B\u0020\u00A0\u2028\u2029]，\S相反
	\w	话语字符，等同于[0-9A-Z_a-z],\W相反
	必须被转义才能匹配字面量的字符：-/[\]^

JavaScript 语言精粹认为：
	\b,(?=),(?!) 不是一个好特性
	一般情况下最好坚持使用贪婪性匹配
	不建议test()方法中对正则表达式使用g标识

支持正则表达式的 String 对象的方法
	search	检索与正则表达式相匹配的值。
	match	找到一个或多个正则表达式的匹配。
	replace	替换与正则表达式匹配的子串。
	split	把字符串分割为字符串数组。
